# plan.md 结构化格式定义

Plan 阶段 OMC 输出的计划必须遵循此格式。Hermes 会在 Phase 1 检查计划是否符合此结构。

```markdown
# Plan: {task description}
> Round: {round_number} | Task: {task_id}

## 任务概述
{一段话说清楚要达成什么}

## 执行步骤

### Step 1: {标题}
- **目标**: {本步骤要达成的目标}
- **操作**: {具体要做什么}
- **预期输出**: {完成后应该看到什么}
- **验收标准**:
  - [ ] {可验证标准1}
  - [ ] {可验证标准2}

### Step 2: {标题}
...

## 文件清单
| 文件路径 | 操作 | 说明 |
|---------|------|------|
| src/main.py | modify | 添加XYZ功能 |
| tests/test_xyz.py | create | 测试XYZ |

## 风险点
1. {风险}: {缓解措施}

## 完成标准
- [ ] 所有Step执行完成
- [ ] 所有验收标准通过
- [ ] 测试套件通过
```

## 设计要点

- **结构化 Steps**：OMX 按 Step 顺序执行，每个 Step 都有明确的"完成定义"
- **文件清单**：明确操作类型（create/modify/delete），减少意外修改
- **验收标准前置**：每步骤都有可验证标准，Hermes 逐条核对
- **风险点**：让 OMX 提前知道哪里容易出问题

## Hermes Plan 检查清单

Phase 1 中 Hermes 检查 plan 时，逐项核对：

- [ ] 有 `## 执行步骤` 和至少一个 `### Step`
- [ ] 每个 Step 有验收标准（`- [ ]` 格式）
- [ ] 有 `## 文件清单`，路径清晰
- [ ] 有 `## 完成标准`
- [ ] Plan 阶段标注不修改文件
